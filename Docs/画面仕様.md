## 設計方針

- **単一ウィンドウ・画面遷移なし。** ドロップ → 設定 → 変換 → 結果確認 → クリア、という一連の流れを 1 画面で完結させる。
  - 一覧・進捗・結果がすべて同じテーブル上で更新されるため、状態がひと目で分かる。
  - 設定（品質・出力先）は常時表示。モーダルやタブに隠さない。
- **一覧 = 作業状態。** ドロップしたファイル群がそのまま「変換キュー兼結果一覧」になる。
- **クリアで初期状態へ戻れる。** 変換後でも一覧を空にして、ドロップ受付の初期表示に戻せる。
- **UI はプラットフォーム非依存を意識。** 将来 iOS 展開に備え、ロジックは Core に寄せ、UI 固有依存を最小化する。

## 画面構成

1 つのメインウィンドウのみ。構成要素は次の 3 エリア。

| エリア | 役割 |
|---|---|
| ツールバー（上部） | 品質スライダー、出力先設定、変換 / 中止、クリア |
| ファイル一覧テーブル（中央） | ドロップ受付、ファイルごとの情報・状態表示。空のときは Empty state |
| ステータスバー（下部） | 全体進捗、件数、合計サイズ・削減率 |

## ウィンドウ全体レイアウト

```
┌──────────────────────────────────────────────────────────┐
│ ツールバー                                                  │
│  品質 [====●====] 0.60   出力先 [同じフォルダ ▾] [選択…]   │
│                                    [ 変換 ]   [ クリア ]   │
├──────────────────────────────────────────────────────────┤
│ ファイル一覧（D&D 受け入れエリア兼用）                       │
│ ┌────┬─────────────┬───────────┬─────────┬────────────┐  │
│ │サムネ│ ファイル名    │ ゲインマップ │ 状態     │ サイズ      │  │
│ ├────┼─────────────┼───────────┼─────────┼────────────┤  │
│ │ 🖼  │ DSC001.jpg  │ ●あり      │ ✓完了   │ 8.2→3.1MB  │  │
│ │ 🖼  │ DSC002.jpg  │ ○なし      │ ◷変換中 │ 6.1→ …     │  │
│ │ 🖼  │ DSC003.jpg  │ ●あり      │ 待機     │ 7.5→ －    │  │
│ └────┴─────────────┴───────────┴─────────┴────────────┘  │
├──────────────────────────────────────────────────────────┤
│ ステータスバー：  2/3 完了   合計 21.8MB → 6.2MB (-72%)    │
└──────────────────────────────────────────────────────────┘
```

### Empty state（一覧が空のとき）

テーブル領域の中央にプレースホルダを表示する。

```
        ┌─────────────────────────────────┐
        │                                 │
        │     ⬇  ここに JPEG / フォルダを    │
        │        ドラッグ&ドロップ           │
        │                                 │
        │   （フォルダ内の *.jpg / *.jpeg を  │
        │      再帰的に読み込みます）         │
        │                                 │
        └─────────────────────────────────┘
```

## 各エリアの仕様

### ツールバー

| コントロール | 仕様 |
|---|---|
| 品質スライダー | 範囲 `0.0`–`1.0`、既定 `0.60`。現在値を数値ラベルで併記。CLI の `-q` に対応。SDR ベース画像の圧縮率に作用し、ゲインマップは生転写のため不変。変換中は無効化 |
| 出力先トグル | 「同じフォルダ」/「指定フォルダ」を切替。既定は「同じフォルダ」 |
| フォルダ選択ボタン | 出力先が「指定フォルダ」のときのみ有効。選択中のパスを近傍に表示。CLI の `-o` に対応 |
| 変換ボタン | 待機行がある場合に有効。押下でバッチ変換を開始。変換中は「中止」表示に切り替わる |
| クリアボタン | 一覧を空にし Empty state に戻す。変換中は無効化 |

- 品質・出力先の設定は次回起動時に復元する（`UserDefaults`）。

### ファイル一覧テーブル

各行が 1 つの入力ファイルを表す。列は以下。

| 列 | 内容 |
|---|---|
| サムネイル | 入力 JPEG の縮小プレビュー |
| ファイル名 | ファイル名（ホバー / ツールチップでフルパス） |
| ゲインマップ | Chip 表示。`●あり`（強調色）/ `○なし`（グレー）。`GainForgeCore.hasGainMap` の判定結果 |
| 状態 | Chip 表示。`待機` / `◷変換中` / `✓完了` / `⚠エラー` / `スキップ`。エラー時はツールチップに理由。**完了行のみ**、Chip の右側にビューワ起動アイコン（例：`􀎶` 虫眼鏡 / `􀏅` 拡大）を併設し、押下で[比較ビューワ](#比較ビューワ別ウィンドウ)を開く |
| サイズ | `変換前 → 変換後`。変換前は読み込み時に確定、変換後は完了時に確定（`-` / `…` で未確定を表現） |

- ドロップは一覧テーブル領域全体で受け付ける（行があってもエリア全体が D&D 対象）。
- 追加時、すでに一覧にある同一パスのファイルは重複追加しない。
- 行の個別操作：右クリックメニュー / Delete キーで「一覧から削除」「Finder で表示」。完了行は「出力 HEIC を Finder で表示」を追加。
- ゲインマップ有無の判定は読み込み時に非同期で行い、判定中はプレースホルダ（例：`判定中…`）を表示する。

### ステータスバー

- 進捗：`完了数 / 総数`（変換中はプログレスインジケータ併記）。
- 集計：完了分の `合計変換前サイズ → 合計変換後サイズ (削減率%)`。
- エラー / スキップがあれば件数を表示（例：`エラー 1 / スキップ 0`）。

## 比較ビューワ（別ウィンドウ）

変換後の結果を、**変換前（入力 JPEG）と変換後（出力 HEIC）を左右に並べて**拡大比較するための専用ビューワ。メインの「単一ウィンドウ・画面遷移なし」方針はファイル処理フローに対するものとし、比較ビューワは**それとは独立した補助ウィンドウ**として扱う（フローの画面遷移には含めない）。

### 位置づけ・起動

- 完了行（`✓完了`）の「状態」セルに表示されるビューワ起動アイコンを押すと開く。
- **ビューワは常に 1 つだけ**（シングルインスタンス）。
  - すでに開いている状態で別の行のアイコンを押すと、**新しいウィンドウは作らず**、既存ビューワの表示内容を押された行の画像に**差し替える**。
  - 既に開いているビューワが背面にある場合は最前面へ移動（`makeKeyAndOrderFront` 相当）し、内容を差し替える。
- 対象行を削除、または一覧をクリアした場合、その行を表示中のビューワは閉じる（または空表示にする）。

### レイアウト

左右 2 ペイン。左＝変換前（入力 JPEG / SDR）、右＝変換後（出力 HEIC）。上部にズーム切替と対象ファイル名を表示する。

```
┌──────────────────────────────────────────────────────────┐
│ DSC001.jpg → DSC001.heic                                   │
│ ズーム [フィット|100%|200%]  [—●────────] 150%             │
├───────────────────────────┬──────────────────────────────┤
│ 変換前（JPEG / SDR）        │ 変換後（HEIC / HDR）           │
│                           │                              │
│        🖼 (pan 可)         │        🖼 (pan 可)            │
│                           │                              │
│   8.2 MB / 4032×3024      │   3.1 MB / 4032×3024  ●ゲインマップ │
└───────────────────────────┴──────────────────────────────┘
```

- 2 ペインは**同じ表示倍率・同じスクロール位置**を共有する。
- 各ペイン下部（または上部）に、サイズ・ピクセル寸法・ゲインマップ有無などの補助情報を表示する。

### ズーム

倍率は**両ペイン同時に**変更する。プリセット（セグメント切替）と任意倍率スライダーの 2 系統を用意し、両者は同じ倍率値を双方向に共有する。

**プリセット（セグメント切替）**

| モード | 挙動 |
|---|---|
| フィット（アスペクトフィット） | ペインに収まるよう全体を表示。既定。画像全体が見えるためパン不要 |
| 100% | 等倍（1 画像ピクセル = 1 表示ポイント）。劣化・ディテールの等倍比較に使う |
| 200% | 2 倍拡大。圧縮ノイズやエッジの精査に使う |

**任意倍率スライダー**

- 範囲は **10%〜800%**、**対数スケール**（100% を中心に拡大・縮小が対称な操作感になる）。現在値を数値ラベルで併記。
- 下限 10%：大きな画像を俯瞰（フィットより引いた表示も可能）。
- 上限 800%：JPEG の 8×8 DCT ブロックが視認でき、圧縮劣化（`-q` の効果）を画素レベルで精査できる。
- プリセットはスライダー上の**スナップ点**として連動する。スライダーを動かすとプリセットの選択表示も追従し、プリセットを押すとスライダーがその倍率へ移動する。
  - 「フィット」は実効倍率（ウィンドウ・画像サイズから算出）でスライダー位置に反映する。手動でスライダーを動かした時点でフィット選択は解除される。

- 100% 以上などペインより画像が大きい場合はパン（後述）で位置を移動する。倍率変更時は、可能ならペイン中心（またはカーソル位置）を基準にズームする。

### パン（ドラッグ移動）と同期

- ペイン内をマウスでドラッグすると表示位置を移動できる。
- **一方をドラッグすると、もう一方も同じ量だけ追従して動く**（左右が常に同じ箇所を映す）。
- 同期は**画像ピクセル座標**を基準とする。変換前後はピクセル寸法が一致する前提（変換は解像度を変えない）。万一寸法が異なる場合は左ペインのピクセル座標を基準に右を合わせる。
- アスペクトフィット時は全体が収まりパンは不要のため、ドラッグは無効（または何もしない）。

### HDR 描画（変換前後の比較）

比較の主目的は **(a) ゲインマップによる HDR 表示効果の確認** と **(b) SDR ベースでの圧縮劣化の確認** の両方。

- 右ペイン（変換後 HEIC）は、ディスプレイが対応していれば **HDR（EDR）で描画**し、ゲインマップが効いた明部の伸びを確認できるようにする。
- 左ペイン（変換前 JPEG）は SDR ベース画像として描画する。ゲインマップ付き JPEG の場合も、変換前の見えとして SDR ベースを表示する。
- HDR 非対応ディスプレイでは、システムのトーンマッピングに従って SDR 表示にフォールバックする。
- 等倍（100% / 200%）比較により、SDR ベースの圧縮劣化（`-q` の効果）も画素レベルで確認できる。

> 注: 旧「未確定事項」の「HDR プレビュー（変換前後の比較表示）の要否と方式」は本ビューワ仕様で確定とする。

## 状態モデル

### ファイル行の状態遷移

```
            読み込み完了
  (追加) ──────────────▶ 待機
                          │  変換開始
                          ▼
                       変換中 ──┬──▶ 完了
                                ├──▶ エラー
                                └──▶ スキップ（出力衝突などで変換しない場合）
```

- ゲインマップ無しの JPEG も **常に変換する**（SDR HEIC 化）。ゲインマップ有無で待機可否は変えない。
  - ゲインマップ無しは「ゲインマップ無しの HEIC」として出力する（CLI の `-f` を常時有効にした挙動に相当）。
- 「クリア」は全行を破棄して初期状態（Empty state）へ戻す。

### 画面全体の状態

| 状態 | 説明 | 変換ボタン | クリアボタン | 設定変更 |
|---|---|---|---|---|
| 空 | 一覧なし。Empty state 表示 | 無効 | 無効 | 可 |
| 待機 | 待機行あり、未変換 | 有効（「変換」） | 有効 | 可 |
| 変換中 | バッチ実行中 | 有効（「中止」） | 無効 | 不可 |
| 完了 | 全行が完了 / エラー / スキップ | 無効（待機行なし） | 有効 | 可 |

- 変換中に新規ドロップされたファイルは「待機」として一覧に追加し、現在のバッチには含めない（次回変換で処理）。

## 主要操作フロー

1. **ドロップ**：JPEG / フォルダをドロップ。フォルダは再帰的に `*.jpg` / `*.jpeg` を収集。各行を「待機」で追加し、サムネ・サイズ・ゲインマップ有無を非同期取得。
2. **設定**：必要に応じて品質スライダーと出力先を調整。
3. **変換**：変換ボタン押下。待機行を順に処理し、行状態とステータスバーをリアルタイム更新。
4. **結果確認**：各行の状態 Chip とサイズ、ステータスバーの集計で結果を把握。エラー行はツールチップで原因確認。完了行の起動アイコンから[比較ビューワ](#比較ビューワ別ウィンドウ)を開き、変換前後を左右で拡大比較する（ビューワは 1 つを使い回し、別行を開くと内容が差し替わる）。
5. **クリア / 追加変換**：クリアで初期化、または追加ドロップして再度変換。

- **中止**：変換中に「中止」を押すと、進行中の 1 件完了後に以降の待機行を中断。未処理行は「待機」のまま残す。

## 出力仕様

- **出力先**：
  - 「同じフォルダ」：入力 JPEG と同じディレクトリに `<元名>.heic` を生成。
  - 「指定フォルダ」：選択フォルダ直下に `<元名>.heic` を生成。
- **ファイル名衝突**：出力先に同名 `.heic` が既存の場合、**連番を付与して回避**（上書きしない）。
  - 例：`DSC001.heic` が存在 → `DSC001_1.heic`、さらに存在 → `DSC001_2.heic` …
- **検算**：書き出し後、ゲインマップ有り入力については `hasGainMap` で埋め込みを検算する（[仕様.md](仕様.md) の落とし穴 6 に準拠）。検算失敗時はエラー扱い。

## エラー表示

- 行単位でエラーを表現（状態 Chip `⚠エラー` ＋ ツールチップに `GainForgeError` のメッセージ）。
- バッチ全体は止めず、エラー行をスキップして次へ進む（1 件のエラーで全体を中断しない）。
- ステータスバーにエラー件数を集計表示。

## 確定事項

- 単一ウィンドウ・画面遷移なし。
- ファイル一覧テーブル中心の UI（ドロップ受付・進捗・結果を兼ねる）。
- 出力先は「同じフォルダ / 指定フォルダ」のトグル切替。
- ファイル名衝突は連番付与で回避（上書きしない）。
- ゲインマップ無しの JPEG も常に変換（SDR HEIC 化）。
- クリアで初期状態に戻れる。
- 変換前後の比較は専用の[比較ビューワ](#比較ビューワ別ウィンドウ)（別ウィンドウ）で行う。シングルインスタンス、左右同期パン、変換後ペインの HDR 描画。ズームはプリセット（フィット / 100% / 200%）＋任意倍率スライダー（10%〜800%、対数スケール、両者連動）。
- 比較ビューワの起動アイコンは完了行の「状態」セルに表示する。
